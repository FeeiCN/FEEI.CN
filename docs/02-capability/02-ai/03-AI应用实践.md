# AI应用实践

## 沟通与研究（Chat&Research）

提示词三要素。

- 设定背景：你的角色？你的目标？这件事的背景？
- 明确任务：你希望采取什么行动？你希望AI帮你干什么？
- 明确规则：希望使用什么风格或语气？是否有示例或者想要的效果？

添加上下文：包括文档、图片、表格、代码等。

人工智能流畅性4D框架。与人工智能有效协作的四个关键点，授权、描述、辨别以及勤勉。

回答不满意，持续沟通迭代。

- **回答过于笼统**。提示信息不足以说明具体情况，应当添加更多的背景信息，包括受众、角色或限制条件等。
- **回复内容过长**。AI在猜测合适的长度，应当更明确告诉它你需要的长度，比如一段文字、100字、一份全面的分析等等。
- **没按照要求格式**。知道你想要什么，但不知道你希望呈现的方式，应当给一些格式示例，或者明确描述结构。
- **看起来正确，但实际时错误的信息**。当涉及具体事实或小众话题时会出现，对于高风险工作，需要独立核实关键事实。可以要求提供引用资料来源或说明其置信度。也可以开启网络搜索功能，确保回复是基于最新信息。

研究不再是单一搜索，而是自主开展多项搜索，这些搜索彼此关联，并基于此确定下一步调查方向。适用于全面理解某个问题，需要整合多个信息源、比较不同的观点。

## 技能（Skill）

**技能本质是可重复方式完成特定任务的工具**。技能包括了指令、脚本和资源文件。指令包含具体的步骤、操作顺序以及每次都要遵循的方法。

**技能的创建也可以通过完全的Chat模式实现**。发起新聊天，告诉AI你想创建什么技能。附上你对这个技能的理解、参考资料、案例等。回答AI的问题。下载你的技能，测试并完善。之后就可以使用甚至分享了。

## 工程研发（AI Coding）

### 基本认知

当AI Coding工具（比如 Claude Code、OpenAI CodeX等）收到自然语言的任务时，会遵循人类开发人员解决问题的类似流程：

- 收集上下文信息。弄清楚问题是指什么，代码库哪个部分受影响，哪些文件与此相关。
- 制定计划。决定如何解决问题，比如修改代码并运行测试来验证修复效果。
- 采取行动。通过更新文件喝运行命令来实际实施解决方案。

[![](https://feei.cn/wp-content/uploads/instructor_a46l9irobhg0f5webscixp0bs_public_1750967940_002_-_What_is_a_Coding_Assistant__02.1750967940100.png)](https://feei.cn/wp-content/uploads/instructor_a46l9irobhg0f5webscixp0bs_public_1750967940_002_-_What_is_a_Coding_Assistant__02.1750967940100.png)

此处和传统对话AI（比如ChatGPT、DeepSeek、Grok、Gemini、Qwen等）模型差异是，传统对话大模型只会告诉你第二部如何干，但它并不能真的去帮你做什么，而AI Coding工具会帮你完成第一步和第三步，会和你项目的真实文件进行交互，包括读取、修改你的代码、文档，运行命令等。这些背后，是专属训练模型能够理解意图，并转换为工具调用，从而能够带来变成能力本质的提升。

- 所谓的工具，包括Agent（运行一个子Agent来解决一个任务）、Bash（运行Shell命令）、Edit（编辑文件）、Glob（基规则读取文件）、Grep（搜索文件）、LS（列文件目录）、MultiEdit（多文件修改）、NotebookEdit/Read（读写Jupyter notebook），以及后续还可能有Read（读取文件）、Todo读写、Web Search&Fetch、Write写文件等。
- 在代码研发中，并不需要太多上层复杂工具，而是底层的文件操作和命令执行，毕竟人类传统研发过程中工具都是基于此建立起来的。其次，工具的核心是在于如何智能组合使用这个工具，从而解决复杂的多步骤问题。

[![](https://feei.cn/wp-content/uploads/instructor_a46l9irobhg0f5webscixp0bs_public_1750967942_002_-_What_is_a_Coding_Assistant__14.1750967942536.png)](https://feei.cn/wp-content/uploads/instructor_a46l9irobhg0f5webscixp0bs_public_1750967942_002_-_What_is_a_Coding_Assistant__14.1750967942536.png)

每个项目会存在大量文件，如何管理上下文尤为重要。比如在Claude Code中，会在最开始工作前，通过/init命令来扫描代码库，创建一个项目总结内容并写入到CLAUDE.md文件中，并在后续每次任务时带上这个文件。

[![|669x376](https://feei.cn/wp-content/uploads/instructor_a46l9irobhg0f5webscixp0bs_public_1750967941_004_-_Adding_Context_05.1750967940882.png)](https://feei.cn/wp-content/uploads/instructor_a46l9irobhg0f5webscixp0bs_public_1750967941_004_-_Adding_Context_05.1750967940882.png)

一些技巧。

- 你想操作某一个文件可通过@文件名，Claude Code/OpenAI CodeX都遵守该逻辑。
- 涉及到报错时，可以直接将报错信息发给AI。
- 涉及到前端时，可以直接将前端页面截图发给AI。
- 如果跑题或做多余你不想它做的事情时，及时打断。
- 如果经常性出行一些重复问题时，让他记住正确的做法。（Claude Code中使用#）
- 长时间对话会导致上下文变得混乱。回到前面对话中，避免在一些细节错误对话中分散注意力。（Claude Code中按两次ESC）

参考：[Claude Code in Action](https://anthropic.skilljar.com/claude-code-in-action)
